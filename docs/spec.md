# 電力センサーモジュール 基板設計仕様書

## 1. 目的
分電盤の各ブレーカーの電流(または電力)をCTクランプで非接触測定し、
ESPHome経由でHome Assistantに取り込む。

## 3. 回路構成の要点

- 3.3VとGNDの間に10kΩ×2(R28, R29)+10µF(C17)でバイアス電圧(1.65V)を生成し、
  LMV358(U2)のボルテージフォロワでバッファして全12chに共有供給(VBIASネット)
- 各chの負荷抵抗は「固定抵抗(R1〜R12)+100Ω多回転トリマー(R13〜R24, 3296W-1-101LF, 25回転)」の直列(2026-07-03改訂)。
  トリマーは大レンジ切替用ではなく**微調整用**と割り切り、レンジの大枠は固定抵抗側で決める:
  - **CN1(高電流用ch, R1=22Ω)**: R3=22〜122Ω。30A/60Aどちらもこの1chで、トリマー位置だけ変えて対応(BOM変更不要)
    - 30A狙い(目標フルレンジ約35A): R3≈61Ω(トリマー約39Ω, 39%位置)
    - 60A狙い(目標フルレンジ約66〜70A): R3≈30〜32Ω(トリマー約8〜10Ω, 8〜10%位置)。低R3側は電流に対する感度が急峻なため、この位置での微調整分解能は相対的に粗くなる(トリマーの選定では解決しきれない物理特性。最終的には`calibrate_linear`のソフト校正で追い込む前提)
  - **CN2〜CN12(15A用ch, R2〜R12=39Ω)**: R3=39〜139Ω。目標フルレンジ約18A(15A×1.2)でR3≈118Ω(トリマー約79Ω, 79%位置)
  - トリマーのワイパー(2番)と未使用端(3番)は接続済み(接触不良時に開放とならないフェイルセーフ)
  - トリマーは旧3266W-1-101(12回転)から3296W-1-101LF(25回転, 同100Ω)に変更(2026-07-03)。回転数を増やすことで、抵抗値を変えずに分解能を約2倍(8.3Ω/回転→4Ω/回転)に改善。パッケージが6.7×4.5mm→9.5×4.8mmに拡大するためPCB上のフットプリント選定に反映すること
- 各chの負荷抵抗と並列に100nF(C1〜C12)のフィルタ
- **負荷抵抗と測定可能な電流レンジの関係**(2026-07-03追記、固定抵抗を2種類に分けた後の版): CT巻数比2000:1、バイアス1.65V中心、クリップ回避のマージンとして許容ピーク電圧1.5Vを見込むと、
  `フルスケール一次電流[A rms] ≈ 2121 / R3[Ω]`。この式に基づき、CN1(固定22Ω、高電流用)はトリマー位置で30A/60A狙いを切り替え、CN2〜CN12(固定39Ω、15A用)は約18Aフルレンジ狙いで固定する(詳細な数値は3節参照)
- **フルスケールを超える電流が流れた場合**: 波形のピークが電源レール(0V/3.3V)でクリップし、`calibrate_linear`の元になる実効値算出が実際より小さく出る(過小評価。ハード破損のリスクは無いが校正がズレる)。目安のフルスケールを常時超えそうなchは、トリマーを回してR3を下げ(感度を落とし)、実際の想定最大電流に対して余裕を持たせること
- CT接続は3.5mmジャック(PJ-320D、4極)。チップ=信号、リング1/リング2/スリーブ=VBIAS(接触冗長化)
- ADS1115はシングルエンドモード、gain設定は測定電流レンジに応じて調整
- `calibrate_linear`フィルタで実測校正(クランプメーターとの比較測定)+トリマーによるハードウェア校正
- ステータスLED(白, KT-0805W)はD0でシンク駆動、1kΩ制限(約0.5mA、控えめな輝度)

## 4. 確定した仕様

### 4.1 測定チャンネル数(2026-07-02改訂)
- 最大12ch(3.5mmジャック×12)
- ADS1115×3個(4ch×3)、I2Cアドレス 0x48/0x49/0x4A

### 4.2 測定する物理量
- 電流(A)のみ実測し、想定電圧(例:100V)×電流で電力(W)を近似する
- 電圧センサー(ZMPT101B等)は追加しない

### 4.3 バイアス回路の設計
- 3.3V/2分圧のバイアス電圧をオペアンプ(ボルテージフォロワ)でバッファリングし、12ch共有の1系統として供給
- オペアンプは単電源・rail-to-rail入出力品(LMV358)を採用
  - 当初候補のLM358は入力同相範囲が保証値でV+−2V(3.3V電源では1.3V)までのため、
    バイアス電圧1.65Vを扱えず不採用(2026-07-02設計レビューで判明)

### 4.4 電源
- USB-C給電(XIAO内蔵レギュレータで3.3V生成)を基本とする
- 外部給電用に3.3V/GNDのスルーホールパッド(TP1/TP2、φ3.2mm)を用意(2026-07-02追加)
  - USB-C給電と外部3.3V給電の同時使用は不可(レギュレータ出力衝突のため)
- 3.3V負荷はADS1115×3+LMV358+分圧・プルアップ・LEDで合計約3mA。
  XIAOのWi-Fi送信ピーク(約300mA)に対し誤差レベルで、内蔵レギュレータで十分

### 4.5 CTクランプ接続コネクタ(2026-07-02改訂)
- 3.5mmステレオジャック(PJ-320D)を採用(SCT-013の3.5mmプラグと直結できるため)
- 当初案のJSTコネクタは、CTクランプ側ケーブルの加工が必要になるため不採用

### 4.6 部品の実装方式
- ADS1115・XIAO ESP32C6ともに基板へ直接ハンダ付け(ICチップ/モジュール直付け、ソケット・ピンヘッダなし)
- トリマー(3296W)はスルーホール品のため手ハンダ

### 4.7 筐体・設置形態
- 独立ケース(3Dプリント等)を想定。DINレール取付は不採用
- 基板サイズの制約は特になし(部品配置に余裕を持たせる)

### 4.8 キャリブレーション運用(2026-07-02改訂)
- 2段構え:
  1. ハードウェア校正: chごとのトリマー(0〜100Ω)でブレーカー容量に応じたレンジ調整
  2. ソフトウェア校正: 個体ごとに実測(クランプメーターとの比較測定)し`calibrate_linear`で調整

### 4.9 I2Cバス
- SDA=D4, SCL=D5、プルアップ10kΩ×2(R26=SDA, R25=SCL)

### 4.10 基板仕様
- 2層基板、JLCPCB発注
- 部品はJLCPCB Basic Part優先で選定(受動部品はすべてBasic化済み。詳細は [bom.md](bom.md))

### 5.2 ハードウェア起因でファームウェアに引き継ぐ事項(回路設計の帰結)

- **ADS1115のgainは4.096固定**(`gain: 4.096`)。バイアス1.65Vを中心に信号が0〜3.3Vまで振れるため、
  gain 2.048(FSR±2.048V)では2.048V超がクリップして正の半波が欠ける。全chで4.096必須
- **サンプリングレートの制約**: ADS1115は最大860SPS。`ct_clamp`の測定窓内で60Hzの1周期あたり
  最大約14サンプルしか取れないため、`sample_duration`は複数周期分(例: 200ms以上)を確保して平均化する。
  12ch(3IC×4ch)はICごとに順次マルチプレクスされるので、全chの更新周期はその分長くなる前提で
  `update_interval`を設計する
- **I2Cは標準速度(100kHz以下)で使う**: プルアップが10kΩのため400kHz(Fast-mode)はエッジが鈍る。
  ESPHomeのデフォルト(50kHz)または100kHzのままにする
- **U5のアドレスは0x4A(ADDR→SDA接続)**: `ads1115:`のaddress指定は 0x48(U3)/0x49(U4)/0x4A(U5)
- **チャンネル対応**: CN1〜4→U3(0x48)のA0〜A3、CN5〜8→U4(0x49)のA0〜A3、CN9〜12→U5(0x4A)のA0〜A3
- **校正は2段階**: 先にトリマー(R13〜R24)でレンジを合わせてから`calibrate_linear`で仕上げる。
  docs/calibration.mdへのトリマー手順追記も必要
- **LED1はD0のシンク駆動(LOW点灯)**: `pin: D0, inverted: true`相当の設定にする
