# 校正(キャリブレーション)について

CTクランプ電流センサーは個体差・回路誤差(バイアス電圧のずれ、負荷抵抗の誤差、CTクランプ自体の誤差)があるため、
出荷前に実測でチャンネルごとに校正する運用とする(spec.md 4.8参照)。

## 校正の考え方

ADS1115から得られる生の電圧値(V)と、実際にCTクランプに流れている電流(A)の関係は
理想的には比例関係だが、以下の理由でオフセットと傾きの両方がずれる。

- バイアス回路(R1/R2 + オペアンプ)の分圧誤差により、無負荷時の中心電圧が理論値(1.65V)からずれる
- 負荷抵抗(R3)の誤差(E24系列の公称値と実測値の差)
- CTクランプ自体の巻数比誤差・線形性

そのため [esphome-sample.yaml](../firmware/esphome/esphome-sample.yaml) の `calibrate_linear` フィルタで
実測に基づく2点以上の対応関係を与え、線形補正する。

```yaml
filters:
  - calibrate_linear:
      - 0.0 -> 0.0    # (ADS1115実測電圧V) -> (実電流A)
      - 1.65 -> 0.0   # 無負荷時のオフセット電圧
      - 2.00 -> 10.0  # 既知負荷での実測点
```

## 校正手順

### 準備するもの

- クランプメーター(実電流の基準値を測定するため)
- 既知の負荷(電気ケトルやドライヤーなど、消費電力が分かっている・調整できる負荷)
- ロガー用PC(Home AssistantのAPI経由、またはESPHomeログでリアルタイム値を確認)

### 手順

1. **無負荷点の記録**
   対象ブレーカーの負荷をゼロにした状態で、ESPHomeログ上の生値(ADS1115電圧、`internal: true` を一時的に外すかログで確認)を記録する。
   これが `calibrate_linear` の1点目(オフセット)になる。

2. **既知負荷点の記録**
   クランプメーターを対象ブレーカーの配線に取り付け、既知の負荷(できるだけ定格に近い電流)を流した状態で、
   - クランプメーターの実測電流値
   - ESPHome側の生値(ADS1115電圧)

   を同時に記録する。

3. **複数負荷点での確認(推奨)**
   負荷を変えて2〜3点程度追加で記録し、線形性を確認する。大きくずれる場合は回路(R3の実測値、バイアス電圧)を見直す。

4. **`calibrate_linear` の更新**
   記録した (生値, 実電流) のペアを [esphome-sample.yaml](../firmware/esphome/esphome-sample.yaml) の該当チャンネルの
   `calibrate_linear` に反映し、ESPHomeを再フラッシュする。

5. **検証**
   校正後、再度クランプメーターと比較し、誤差が許容範囲内(目安: ±5%程度)に収まっていることを確認する。

## 個体ごとの管理

- 基板・CTクランプの組み合わせごとに校正値は異なるため、モジュール単位(またはブレーカーごと)に校正記録を残すこと
- 校正記録には「日付」「対象チャンネル」「生値と実電流の対応点」「使用したクランプメーター機種」を残すと再校正時に比較しやすい
- CTクランプを交換した場合、負荷抵抗を変更した場合は再校正が必要

## 保留事項

- update_interval(サンプリング間隔)や sample_duration の最適値は実運用しながら調整する(spec.md 5.1参照)
- 電圧は実測せず想定値(例: 100V)を用いた近似のため、電力(W)の絶対値には想定電圧との誤差がそのまま乗る
